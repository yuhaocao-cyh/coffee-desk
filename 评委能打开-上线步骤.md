# 让评委用手机打开你的网站（从 localhost 变成公网链接）

你现在用的是 **`http://localhost:3000`**，这只在你自己电脑上有效。  
要让**评委在现场用手机浏览器打开**，需要把网站**部署到公网**（下面用 **Vercel**，免费、和 Next.js 最省事）。

---

## 你会得到什么？

部署成功后，你会得到一个类似这样的地址：

`https://你的项目名.vercel.app`

把这个链接发给评委，或印在 PPT / 易拉宝上，**手机用 4G/5G 或现场 Wi‑Fi 都能打开**（不要求和你在同一个 Wi‑Fi）。

---

## 第 0 步：先确认本机能跑通

在项目文件夹里：

```powershell
cd "$env:USERPROFILE\Desktop\强农杯\jiaoyi-web"
npm run dev
```

本机 `localhost` 没问题再继续，避免「上线了才发现代码有错」。

---

## 第 1 步：把代码放到 GitHub 上

Vercel 一般从 GitHub 拉代码自动部署。

1. 打开 <https://github.com> 注册/登录。  
2. 右上角 **New repository**，新建一个空仓库（例如 `coffee-desk`），**先不要**勾选自动添加 README（空仓库最好推）。  
3. 在你电脑上打开 **PowerShell**，进入项目目录：

```powershell
cd "$env:USERPROFILE\Desktop\强农杯\jiaoyi-web"
```

4. 若项目**还没有** git 仓库，执行：

```powershell
git init
git add .
git commit -m "Initial commit"
```

5. 把 GitHub 上新建仓库页面里那两行「push 已有仓库」的命令抄下来执行（里面带你的仓库地址），一般是：

```powershell
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

> 若 `git` 提示未安装：先安装 [Git for Windows](https://git-scm.com/download/win)，装完重新打开 PowerShell。

---

## 第 2 步：用 Vercel 一键部署

1. 打开 <https://vercel.com> ，用 **GitHub 登录**。  
2. **Add New… → Project**，选择你刚推上去的仓库 `jiaoyi-web`（或你起的名字）。  
3. **Framework Preset** 保持 **Next.js** 即可。  
4. 先**不要**点 Deploy——先做下一步「环境变量」。

---

## 第 3 步：在 Vercel 里填 Supabase 密钥（和本机 `.env.local` 一样）

在 Vercel 创建项目页面里找到 **Environment Variables**，添加两行（名字必须一致）：

| Name | Value |
|------|--------|
| `NEXT_PUBLIC_SUPABASE_URL` | 你在 Supabase 复制的 Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 你在 Supabase 复制的 anon public key |

然后点 **Deploy**。等 1～2 分钟，出现 **Visit** 或预览链接，那就是你的**公网地址**。

---

## 第 4 步：让 Supabase 允许「线上域名」登录（必做，否则手机登录会失败）

1. 打开 Supabase 控制台 → **Authentication** → **URL Configuration**。  
2. **Site URL** 改成你的线上地址，例如：  
   `https://你的项目名.vercel.app`  
3. **Redirect URLs** 里**新增**一行（不要删 localhost，可以两行都保留）：  

   `https://你的项目名.vercel.app/auth/callback`

4. 点 **Save**。

---

## 第 5 步：现场怎么给评委用最省事？

**工作台 `/desk` 需要登录**才能看数据。现场最省事的做法二选一：

- **方案 A（推荐）**：你提前注册好 **一个演示账号**，把 **邮箱 + 密码** 打在 PPT 最后一页或口头告诉评委「点登录输入这个」。  
- **方案 B**：评委各自用手机 **现场注册**（依赖现场网络，会慢一点）。

若只想「打开就能看界面、不要求登录」，可以用首页的 **本机版** 思路——但本机版数据在浏览器里，**不适合多人传阅**；公网演示仍建议用 **云端工作台 + 演示账号**。

---

## 第 6 步：手机自测清单（答辩前一天做）

用手机 **关掉 Wi‑Fi、只用流量** 打开：

`https://你的项目名.vercel.app`

- [ ] 首页能打开  
- [ ] 能 **登录**（或演示账号能登录）  
- [ ] **工作台** 能打开、能提交一条测试数据  

若只有连 Wi‑Fi 才能打开、换流量打不开，多半是 DNS 或公司网限制，答辩当天尽量用 **手机流量** 测一遍。

## 常见问题

**Q：提示「GitHub 仓库不包含请求的分支或提交引用 / 仓库是空的」怎么办？**

Vercel 的意思是：**连上的那个 GitHub 仓库里没有可用的代码或分支**（常见是**仓库是空的**，或分支名对不上）。

按下面自检（用浏览器打开你的 GitHub 仓库主页）：

1. **仓库里能不能看到 `package.json`、`app` 文件夹等？**  
   - 若页面提示「Quick setup」、只有 README、**没有你的 Next 代码**：说明**还没把本地代码 push 上去**。  
   - **处理**：在你电脑上安装 [Git for Windows](https://git-scm.com/download/win)，装完**重新打开** PowerShell，进入 `jiaoyi-web` 目录执行：

```powershell
cd "$env:USERPROFILE\Desktop\强农杯\jiaoyi-web"
git init
git add .
git commit -m "deploy: initial"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

把 `你的用户名/你的仓库名` 换成 GitHub 上真实的地址。push 成功后，刷新 GitHub 网页应能看到完整项目文件，再回到 Vercel **重新 Import / Redeploy**。

2. **分支是不是叫 `main`？**  
   - Vercel 默认拉 **`main`**。若你只有 **`master`**，要么在 Vercel 项目 **Settings → Git → Production Branch** 改成 `master`，要么本地执行 `git branch -M main` 再 push。

3. **Vercel 是否连错了仓库？**  
   - 在 Vercel 导入页确认选中的就是**你刚 push 过代码的那个仓库**，不是 GitHub 上另一个空仓库。

**Q：提示「项目已经存在，请使用新名称」怎么办？**  

这是 **Vercel 上的「项目名称」和别人撞了**（全站不能重名），和你的 GitHub 仓库名不是一回事。

- **正在新建项目时**：在创建页面找到 **Project Name**（项目名称），改成**全世界不太可能重复**的名字，例如：  
  `qiangnong-kafei-你的英文名-2026` 或 `coffee-desk-cyh-0520`  
  再点 Deploy。你的网址会变成 `https://你改后的名字.vercel.app`。
- **已经建过一个同名项目**：登录 Vercel → **Dashboard** 里看是不是已经有一个项目占用了这个名字。  
  - 若那是你之前试错的：可以进该项目 **Settings → General → Delete Project** 删掉再建；  
  - 或保留旧项目，**新建时用另一个名字**即可。

改完名字后，记得回到 **Supabase → Authentication → URL Configuration**，把 **Site URL** 和 **Redirect URLs** 里的域名改成**新的** `https://新名字.vercel.app`（以及 `/auth/callback` 那一行）。

**Q：部署后改代码怎么办？**  
把新代码 `git push` 到 GitHub，Vercel 会自动再部署一版（一般 1～2 分钟）。

**Q：链接太长？**  
可以答辩后再研究自定义短域名；现场直接浏览器收藏或二维码生成器把 `https://xxx.vercel.app` 做成二维码也行。

**Q：不想用 GitHub 行不行？**  
可以，用 Vercel CLI 从本机部署，但步骤更多；学生比赛用 GitHub + Vercel 最常见。
