# 云豆集 · 深化版上线步骤

> 假设你的网站已经部署在 Vercel（https://coffee-desk-two.vercel.app/），以下步骤将更新为新增功能。

---

## 第 1 步：在 Supabase 执行迁移 SQL

按顺序在 Supabase → SQL Editor 中执行以下 SQL 文件：

### 1.1 订单状态迁移
打开并执行 `supabase/migrations/002_order_status.sql`
- ✅ 为 purchase_orders 增加 status 字段（pending/confirmed/shipped/completed/cancelled）
- ✅ 增加状态时间戳字段

### 1.2 库存迁移
打开并执行 `supabase/migrations/003_inventory.sql`
- ✅ 为 coffee_lots 增加 stock_kg 字段
- ✅ 为 5 个示例批次填充初始库存

### 1.3 用户角色迁移
打开并执行 `supabase/migrations/004_user_roles.sql`
- ✅ 新建 user_profiles 表（角色系统）
- ✅ 更新 RLS 策略（管理员权限）

### 1.4 索引迁移
打开并执行 `supabase/migrations/005_dashboard_indexes.sql`
- ✅ 性能索引

---

## 第 2 步：设置管理员账号

在 Supabase → SQL Editor 中执行（替换为你自己的用户 UUID）:

```sql
insert into public.user_profiles (id, display_name, role)
values ('你的用户UUID', '管理员', 'admin');
```

> 如何获取 UUID？先注册一个账号并登录，然后在 Supabase → Authentication → Users 中可以看到你的用户 UUID。

## 如果遇到问题

1. **页面白屏或报错** → 检查 Vercel 部署日志（Deployments → 最新一次 → View logs）
2. **数据库查询失败** → 确认 4 个 SQL 迁移都已按顺序执行
3. **权限不足** → 确认已执行 INSERT INTO user_profiles 设置管理员
4. **图片不显示** → 确认 next.config.ts 中的 remotePatterns 包含所用域名
