# 云豆集（jiaoyi-web）保姆级部署教程

> **觉得下面太麻烦？** 可以先用 **零配置本机版**：项目里运行 `npm run dev` 后，浏览器打开首页，点 **「本机版（零配置）」** 或访问 `http://localhost:3000/benji` —— 不注册、不配数据库，数据存在本机浏览器。等需要多人长期再用本文云端部分即可。

按顺序做，**不要跳步**。做完大约 **15～25 分钟**（含等待 Supabase 项目启动）。

---

## 你需要提前准备什么？

1. **一台能上网的电脑**（Windows 没问题）。
2. **一个常用邮箱**（用来注册 Supabase、用来登录你的网站）。
3. **Node.js** 已安装（若不确定：按 `Win + R`，输入 `cmd` 回车，黑窗口里输入 `node -v`，能显示版本号比如 `v20.x` 就行）。  
   - 若没有：打开 <https://nodejs.org/zh-cn> 下载 **LTS** 安装，一路下一步即可。

---

## 第一部分：在 Supabase 上开一个「云端数据库」

### 第 1 步：注册并登录

1. 浏览器打开：**<https://supabase.com>**  
2. 右上角点 **Sign in**（或 **Start your project**）。  
3. 用 **GitHub** 或 **邮箱** 注册都可以，按页面提示完成登录。

### 第 2 步：新建项目（New project）

1. 登录后，点 **New project**（新建项目）。  
2. **Organization**：一般选默认即可。  
3. **Name**：随便起，例如 `coffee-desk`。  
4. **Database Password**：自己设一个**强密码**（请用密码管理器或记事本**保存好**，以后很少用到，但丢了会麻烦）。  
5. **Region**：选离中国近的（例如 **Singapore** 新加坡），一般更快。  
6. 点 **Create new project**。  
7. **等待 1～3 分钟**，页面出现表盘、菜单，不再一直转圈，就算好了。

> 若一直卡住：换网络、换浏览器，或稍后再试。

---

## 第二部分：把「网址」和「钥匙」抄到你的电脑项目里

你的网站要靠这两样东西连接云端：

- **Project URL**（项目网址）  
- **anon public key**（匿名公钥，可以放在网页里那种）

### 第 3 步：在 Supabase 里找到这两项

1. 在 Supabase 左侧点 **齿轮图标 Project Settings**（项目设置）。  
2. 左侧点 **API**。  
3. 你会看到：  
   - **Project URL**：形如 `https://xxxx.supabase.co`  
   - **Project API keys** 里有一项 **anon public**：一长串以 `eyJ` 开头的字符  

4. **分别复制**这两段（建议先粘到记事本里备用，别发给别人）。

### 第 4 步：在你电脑上创建 `.env.local` 文件

1. 打开文件夹（就是你 Cursor 里的这个路径）：  
   `C:\Users\caoyuhao\Desktop\强农杯\jiaoyi-web`  
2. 找到 **`.env.local.example`**（文件名以点开头；若资源管理器里看不见，在 Cursor 左侧文件树里一定能看到）。  
   - **复制**该文件，把副本重命名为 **`.env.local`**（注意开头有个点，且没有 `.example` 后缀）。  
3. 用记事本 / VS Code / Cursor 打开 **`.env.local`**，改成类似下面（**换成你自己的**，不要照抄示例里的假地址）：

```env
NEXT_PUBLIC_SUPABASE_URL=https://你的项目ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJI...（整段粘贴）
```

4. **保存文件**。

> **重要**：`.env.local` 不要发到网上、不要发给陌生人。  
> **重要**：改完 `.env.local` 后，下一步若网站已在运行，需要 **关掉黑窗口再重新 `npm run dev`** 才会读到新配置。

## 第三部分：在 Supabase 里「建表 + 放几条示例豆子」

### 第 5 步：打开 SQL 编辑器

1. Supabase 左侧点 **SQL Editor**。  
2. 点 **New query**（新建查询）。

### 第 6 步：粘贴并执行整份 SQL

1. 在你电脑上，用 Cursor 打开这个文件（你已经打开过）：  
   `jiaoyi-web\supabase\migrations\001_init.sql`  
2. **全选**文件里的所有内容（`Ctrl + A`），**复制**（`Ctrl + C`）。  
3. 回到 Supabase 的 **SQL Editor** 大空白框里，**粘贴**（`Ctrl + V`）。  
4. 右下角点 **Run**（或 **RUN**）。  

### 第 7 步：怎么算成功？

- 下方出现 **Success** / **成功** 一类提示，且没有大片红色报错。  
- 左侧点 **Table Editor**，能看到两张表：  
  - **`coffee_lots`**（里面有 5 行示例豆子）  
  - **`purchase_orders`**（一开始是空的，正常）

若报错：

- 若提示 **already exists** 之类：多半你执行过第二次。一般**不影响使用**；若 `coffee_lots` 里数据重复了，可以以后再清理，先继续往下走。  
- 若完全看不懂英文：把报错**全文复制**到记事本，发给我或发给你会英文的同学看。

## 第四部分：把登录规则调成「适合新手测试」

### 第 8 步（强烈建议）：关闭「必须邮箱验证才能登录」

1. Supabase 左侧点 **Authentication**。  
2. 点 **Providers** → 找到 **Email**。  
3. 找到 **Confirm email**（确认邮箱）选项：  
   - **开发自己玩**：把它 **关掉（Disable）**，保存。  
   - 这样你注册后可以**立刻登录**，不用先去邮箱点链接。

> 以后真要对外上线，再考虑打开邮箱验证。

### 第 9 步（建议）：配置登录成功后的跳转网址（本地开发）

1. 仍在 **Authentication** 里，点 **URL Configuration**（网址配置）。  
2. **Site URL** 填：`http://localhost:3000`  
3. **Redirect URLs**（重定向网址列表）里**新增一行**：  

   `http://localhost:3000/auth/callback`

4. 点 **Save**。

## 第五部分：在你电脑上启动网站

### 第 10 步：打开终端并进入项目文件夹

1. 按 `Win + R`，输入 `powershell`，回车。  
2. 复制下面这一行，在蓝色窗口里**右键粘贴**后回车（路径按你的实际桌面用户名，一般不用改）：

```powershell
cd "$env:USERPROFILE\Desktop\强农杯\jiaoyi-web"
```

3. 若提示「找不到路径」：说明你的文件夹不在桌面，请用资源管理器打开 `jiaoyi-web` 文件夹，在地址栏输入 `powershell` 回车（会在该文件夹打开终端）。

### 第 11 步：安装依赖（第一次才需要）

```powershell
npm install
```

等待跑完，没有大片红色 `ERR!` 即可。

### 第 12 步：启动

```powershell
npm run dev
```

看到类似：

`Ready on http://localhost:3000`

就用浏览器打开：**<http://localhost:3000>**

## 第六部分：自己走一遍「注册 → 登录 → 登记」

### 第 13 步：注册账号

1. 打开首页后，点 **注册**（或 **免费注册**）。  
2. 填邮箱 + 密码（密码至少 6 位）。  
3. 提交。

### 第 14 步：进入工作台

1. 点 **工作台**（或首页相关按钮）。  
2. 若提示要先登录：用刚注册的邮箱登录。  
3. 进入后你应该能看到：  
   - 多张「豆子卡片」  
   - 右侧「新建登记」表单  
   - 下方「全员登记动态」

### 第 15 步：提交一条测试数据

1. 表单随便填：采购方、电话、数量等。  
2. 点 **提交到云端**。  
3. 下方列表里出现一条，即成功。  
4. 再注册**另一个邮箱**登录，看能不能看到上一条（能的话，多人共用就 OK）。

## 常见问题（照着排查）

### Q1：打开网站提示要配置 Supabase / missing env

- **原因**：`.env.local` 没放对位置、或没保存、或变量名写错。  
- **处理**：确认文件在 `jiaoyi-web` 根目录，文件名是 `.env.local`，两行 `NEXT_PUBLIC_...` 都填了。然后**关掉** `npm run dev` 再重新运行。

### Q2：工作台提示读表失败 / 表不存在

- **原因**：没执行 `001_init.sql` 或执行失败。  
- **处理**：回到第三部分重做；在 **Table Editor** 确认有 `coffee_lots`。

### Q3：注册后登录不了 / 要验证邮箱

- **原因**：Confirm email 还开着。  
- **处理**：第四部分第 8 步关掉；或去邮箱点验证链接。

### Q4：`npm run dev` 说不是内部命令

- **原因**：没装 Node 或没重启终端。  
- **处理**：安装 Node LTS，关掉所有终端窗口再开。

### Q5：图片加载很慢或裂图

- **原因**：示例图用的是 Unsplash 外链，网络不好时会慢。  
- **处理**：不影响功能；以后可把 `coffee_lots` 表里的 `image_url` 换成你们自己的图。
